# Research Examples

## Strong Fit
- JD: Python + LLM + RAG + FastAPI
- Good research output:
  - choose the most AI/ML-heavy track
  - tighten the summary toward LLM delivery impact
  - add grounded keywords like LLM, RAG, FastAPI
  - strengthen 1-2 recent bullets that already point toward AI work

## Okayish Fit
- JD: cloud platform engineering with some AI adjacency
- Good research output:
  - choose the strongest cloud/data-adjacent track
  - keep the summary realistic
  - add only grounded platform keywords
  - target only one recent experience section
  - call out missing exact-cloud requirements as gaps

## Bad Pattern
- inventing direct GCP hands-on work from AWS-only evidence
- asking for edits across 4+ sections
- rewriting older irrelevant roles in detail
- claiming tools that are not supported by the resume track
