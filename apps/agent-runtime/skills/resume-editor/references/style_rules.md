# Resume Style Rules

## Tone
- Confident and specific
- No hype language
- No generic buzzword stacking
- No recruiter-style marketing copy

## Summary
- 2 to 4 lines max
- Lead with role fit, technical depth, and delivery evidence
- Prefer concrete phrasing such as `production LLM systems`, `Python backend`, `RAG pipelines`, `cloud ML deployment`
- Avoid empty claims like `results-driven`, `dynamic professional`, `seasoned leader` unless the next words immediately ground them

## Skills
- Keep it dense and scan-friendly
- Prefer grouped technical phrases over long prose
- Reorder high-signal skills upward; do not bloat the list
- Do not add a skill if the selected resume track does not support it

## Experience Bullets
- One idea per bullet
- Start with a strong action verb
- Preserve tense consistency
- Prefer evidence + impact + technical mechanism
- Keep bullets believable; do not inflate ownership

## Metrics
- Keep original metrics if already grounded
- Add metrics only if they are already present in source text or clearly derivable from the source content
- Never invent percentage gains, latency reductions, or cost savings

## Truth Boundary
- Exact cloud/platform wording must remain faithful to source evidence
- Do not rewrite adjacent knowledge as direct hands-on delivery
- Do not convert experimentation into production ownership unless the source already supports that claim
