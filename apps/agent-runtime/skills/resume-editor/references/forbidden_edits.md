# Forbidden Edits

## Hard No
1. Claiming GCP delivery when only AWS evidence exists
2. Claiming Azure deployment when the selected track shows no Azure work
3. Replacing Java/.NET-heavy job alignment with false Python-first ownership
4. Adding certifications, patents, or publications not present in source material
5. Rewriting old unrelated roles just to force keyword density

## Examples of Bad Edits
### Bad
`Deployed end-to-end ML platforms on GCP using Vertex AI`

Why bad:
- unsupported if the selected track does not contain that claim

### Bad
`Led enterprise platform modernization across Azure and GCP`

Why bad:
- broad invented cloud ownership

### Bad
`Architected multi-agent systems for financial services clients`

Why bad:
- not acceptable unless the source role already supports that exact domain and level of ownership

## Rule
If you are unsure whether the selected track supports a stronger statement, do not write it.
