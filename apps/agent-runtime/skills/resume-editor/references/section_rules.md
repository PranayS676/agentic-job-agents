# Section Rules

## Summary
- Reframe for the target role using grounded keywords
- Mention the strongest real specialization from the selected track
- Do not claim exact stack ownership if it is a hard gap

## Skills
- Reorder first, add second
- Add only under-emphasized skills that already appear in the selected track or clearly in the target section text
- Prefer exact JD terminology when supported

## Experience
- Edit only the selected `experience_*` section
- Keep the employer, title, location, and dates intact
- Limit substantive edits to 1-2 bullet improvements
- Do not rewrite prior roles outside the selected section

## Non-Editable Areas
- Contact information
- Education
- Certifications
- Old unrelated roles
- Project lists outside the targeted experience section
