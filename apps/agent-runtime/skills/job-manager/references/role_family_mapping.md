# Role Family Mapping

Direct fit:
- AI engineer
- ML engineer
- generative AI
- LLM
- RAG
- prompt engineering
- agentic / agents
- model serving / inference

Adjacent in-scope fit:
- MLOps
- ML platform
- data platform
- Python backend
- platform engineer
- cloud platform engineer
- GCP / AWS platform engineer
- SRE / DevOps
- integration engineer
- cloud architect / solution architect

Keep adjacent roles in scope when they show Python, cloud-native systems, AI enablement, Kubernetes, Terraform, CI/CD, data pipelines, or observability.
Default these adjacent families to `okayish`, not `fit`, unless the role is explicitly AI-platform-heavy or directly centered on Python/AI delivery.

Reject:
- clearly unrelated non-technical or non-overlapping stacks
