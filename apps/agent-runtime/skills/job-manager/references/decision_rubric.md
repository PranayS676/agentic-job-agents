# Decision Rubric

Strong positives:
- AI / ML / LLM / RAG / agent title or stack
- Python plus cloud, backend, data-platform, MLOps, or observability overlap
- concrete technical scope
- C2C, project-based, or W2

Keep in scope:
- cloud-platform, AI enablement, MLOps, SRE, integration, or data-platform roles with real Python/cloud/platform overlap
- roles where seniority is higher than the candidate's years but the technical match is still strong

Strong negatives:
- irrelevant role family
- weak technical detail
- spam / bulk blast with no usable role information
- stack mismatch with little Python, AI, ML, platform, or data relevance

Bias:
- prefer recall over excessive strictness for technically aligned roles

Decision target:
- `fit` for direct AI/ML/Python strengths and clear high-confidence matches
- `okayish` for adjacent cloud/data/platform/backend roles with meaningful overlap
- `reject` for true mismatch or low-signal spam
